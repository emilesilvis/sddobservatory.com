SDD Observatory

# Purpose

Track spec-driven development frameworks and real-world projects over time to evaluate how effective different approaches are in practice.

# Framework directory

Each framework page includes:

* Name and website
* Core approach
* Workflow
* Supported tools
* Maturity
* Strengths and limitations
* Example repositories

# Project tracker

Each project page includes:

* Repository and framework used
* Project size and age
* Spec structure
* Development activity
* Spec-to-code drift
* Defects and rework
* Maintenance outcomes
* Timeline of significant changes

# Comparisons

Allow users to compare frameworks and projects by:

* Adoption
* Activity
* Spec freshness
* Delivery speed
* Defect rate
* Maintenance burden

# Data sources

Start with public GitHub repositories, framework documentation, maintainer submissions, and manually verified case studies.
